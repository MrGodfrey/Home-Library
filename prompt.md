
现在我的需求是建立一个家中藏书的管理系统，具体要求如下：

1. 核心信息展示
   书籍条目需要包含以下信息：标题、作者、出版社、年份，以及 ISBN 号。此外，系统需要显示书籍当前的状态（是否在家中）。

2. 地点分类与管理
   由于我有两个家（成都和重庆），页面需要支持按地点分类展示书籍。我需要能够一眼看到全部藏书，并清晰地知道某本书是在成都还是在重庆。

3. 搜索与自动录入功能
   (a) 搜索功能：可以快速查询某本书是否在家中。
   (b) 扫码录入：网站需要集成一个扫码的小功能，通过扫描 ISBN 号，连接网上 API 自动获取书籍信息并填充到网站上，无需手动录入。

4. 基础管理功能
   系统需要具备完整的增删改查功能。例如，当我不再持有某本书时，点一下即可从系统中删除。

5. 界面与适配
   (a) 展示方式尽可能简单化，可以参考类似豆瓣的展示风格。
   (b) 页面需要做好响应式适配，确保在大屏幕和手机上都能正常使用。

6. 技术架构与部署
   (a) 数据存储：由于这是一个动态网站，需要存储数据并进行计算。我希望将数据和计算资源全部托管在 Cloudflare 上。
   (b) 服务选择：请根据我的需求，告诉我应该使用 Cloudflare 的哪些具体服务（利用其免费资源即可）。
   (c) 域名：我目前已经有一个现成的域名可以使用。

(d) 用户规模与登录系统：
   我目前的用户规模只有两个人，所以你还需要为我配置一个用户登录系统。这个部分当然也可以利用 Cloudflare 的加密系统来保证安全性。当然，也要避免反复的登录。所以你可以把登录状态的保存时间设置得长一点。或者说，这个加密也不用 Cloudflare 那么严格的加密，是吧？

你给我搞一个类似于这种：在 Cloud 里面搞一个数据库之类的，然后可以把用户名和密码手动加进去。

我最终使用的方式应该是在 iPhone 上面。我把这个网站添加到主页之后，它的使用方式应该有点像 App 一样：

1. 点开页面后可以查看当前已有的藏书
2. 搜索某本书是否在库中
3. 也可以添加新书

这些功能都必须要给我实现。
